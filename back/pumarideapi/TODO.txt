[ Problema 1 ]

Actualmente, cada solicitud a /lines/ está guardado en el base de datos.
Desde ese instante cualquiera persona puede hacer match con esa ruta.

Entonces
1. El usuaro no necesiarmante quiso agregarlo como ruta
2. Es posible que el usuario ahora quiere borrar la ruta
3. Si dos hacen match y están de acuerdo, y no hay espacio para
	más gente, deberiamos esconder la ruta como que ya es
	tomado aunque todavía es válida

Como arreglar esto
1. Solamente guardar la ruta si el solicitud viene con algún argumento
	que dice explicitamente que quiere guardarlo
2. Crear un API que deja a los usuarios borrar rutas

Nota que
1. Para borrar idealmente un usuario necesita ser ligado y
	autentificado que es su ruta

Tecnicamente
1. Estamos guardando en ambos sqlite y posgres, necesitamos hacer
	solamente uno

[ Problema 2 ]

No hay un manera para un usuario de buscar todos las rutas que ese usuario
ha creado

Como arreglar esto
1. Ligar cada ruta con el usuario que lo creó
2. Crear un API que deja al usuario cargar sus rutas

[ Problema 3 ]

Como que no son ligados las rutas con los usuarios cuando hace match
no hay manera de mostrar con quien hizo match

Como arreglar esto
1. Ligar cada ruta con el usuario que lo creó
2. Actualizar el API de match para regresar este información
